# 用户录音功能设计文档

## 需求概述

在宝宝白噪音小程序中添加用户自定义录音功能：
- 用户可以录制自己的音频
- 文件大小必须控制在小程序限制内
- 录音存储在用户本地
- 最长录制30秒
- 新增独立 tab 栏显示录音列表
- 支持重命名和删除管理

## 技术方案

### 编码参数选择（方案1）

| 参数 | 值 | 说明 |
|------|-----|------|
| 采样率 | 16000 Hz | 足够语音录制，文件小 |
| 声道 | 单声道 | 文件大小减半 |
| 码率 | 64 kbps | 平衡音质和大小 |
| 时长 | 30秒 硬性限制 | 最大大小 ~240KB，绝对安全 |

### 大小计算

```
大小 = 码率 × 时长 ÷ 8（字节）
64000 bit/s × 30 s ÷ 8 = 240000 bytes ≈ 240KB
```

远小于小程序限制，完全安全。

### 数据结构

#### 录音项存储格式
```javascript
{
  id: string,            // 唯一ID (时间戳)
  name: string,          // 用户命名
  filePath: string,      // 本地文件路径
  duration: number,      // 实际时长（秒）
  createTime: number     // 创建时间戳
}
```

#### 存储位置
- 音频文件：微信小程序本地文件系统 `wx.getFileSystemManager()`
- 元数据：存在本地存储 `wx.getStorageSync()`

### 功能流程

1. **进入页面**
   - 从存储读取录音列表
   - 显示列表，空状态提示"还没有录音"

2. **开始录音**
   - 用户点击"+"按钮
   - 调用 `wx.getRecorderManager()` 开始录音
   - 显示倒计时/进度条
   - 30秒到自动停止

3. **完成录音**
   - 录音停止，获得临时文件路径
   - 弹出输入框让用户命名
   - 移动文件到持久化存储
   - 保存元数据到存储
   - 刷新列表

4. **播放录音**
   - 用户点击录音项
   - 通过 `audio-manager` 播放，复用现有播放逻辑
   - 和内置声音一样支持循环播放

5. **管理操作**
   - 点击"更多"按钮（或长按）弹出操作菜单
   - 选项：重命名 / 删除
   - 重命名：弹出输入框修改名称
   - 删除：确认后删除文件和元数据

### 音频播放集成

复用现有 `audio-manager.js` 播放逻辑：
- 用户录音路径就是本地文件路径
- `innerAudioContext.src = filePath` 可直接播放
- 无需修改现有播放代码

### 权限处理

- 首次录音需要申请麦克风权限
- 权限被拒需要提示用户打开权限设置
- 错误处理展示在界面上

### UI设计

- 遵循现有设计风格
- 列表使用和sound-card类似的卡片样式
- 录音按钮大圆角，红色方便点击
- 录音中显示倒计时动画

## 风险控制

1. **存储超限处理**
   - 限制最多保存10条录音（可调整）
   - 提醒用户删除不需要的录音

2. **文件系统错误处理**
   - 捕获所有文件操作异常
   - 展示友好错误提示

3. **后台录音**
   - 页面隐藏时可继续录音
   - 停止时正确处理文件
